# Agentic Studio - Comprehensive Wiki

## Overview
Agentic Studio is a sleek, modern Next.js web application designed for managing and collaborating with AI agents. It serves as an immersive dashboard to interact with multiple AI personas (agents), track their tasks dynamically, and review the code they generate or modify.

## Architecture & Tech Stack
The project is built on a cutting-edge web stack focusing on performance and aesthetics:
- **Framework**: Next.js 16 (React 19) utilizing the App Router.
- **Styling**: Tailwind CSS v4 (using the new `@tailwindcss/postcss` setup).
- **Animations & UI**: Framer Motion for smooth transitions, Lucide React for consistent iconography.
- **Language**: TypeScript for strong typing and developer experience.

## Core Features & Components

### 1. Dashboard Client (`src/components/DashboardClient.tsx`)
The central orchestrator of the application. It maintains the unified state for:
- **Messages**: The chat history between the user and various AI agents.
- **Kanban Board**: The column and task states (e.g., Requirements, Architecture, Implementation).
- **Code Files**: The virtual file system of generated code displayed to the user.
- **Orchestration**: Simulates a workflow via `/api/orchestrate`, reading a Server-Sent Events (SSE) stream to dynamically update messages, task statuses, and code files in real-time.

### 2. Agent Chat (`src/components/AgentChat.tsx`)
A real-time conversational interface where the user interacts with a team of AI agents. 
- **Personas**: Includes specialized roles like *System Architect*, *Product Manager*, and *Lead Developer*.
- **UI Details**: Features a dynamic glass-morphism panel, typing indicators, and distinct visual styling (colors and icons) for each agent role to make conversations easy to follow.

### 3. Kanban Board (`src/components/KanbanBoard.tsx`)
A live workflow tracker that visually represents the progress of the AI agents.
- **Stages**: Automatically updates tasks moving through columns such as *Requirements*, *Architecture*, and *Implementation*.
- **Status Indicators**: Uses icons to represent 'todo', 'in-progress' (spinning loader), and 'done' (checkmark) states, giving immediate feedback on sprint progress.

### 4. Code Viewer (`src/components/CodeViewer.tsx`)
An integrated environment to review code generated by the agents.
- **Features**: Multi-tab interface for navigating different files, syntax highlighting (customized via tailwind utility classes), line numbers, and a one-click copy-to-clipboard function.

## State Management and Data Flow
The application currently uses React's `useState` for state management, simulating a complex unified state. The state flows top-down from `DashboardClient` to the respective components. The system is designed to consume a streaming API (`/api/orchestrate`) which pushes structured JSON updates (message, typing, kanban, code) that are parsed and applied to the client state, driving dynamic UI updates without page reloads.

## Setup and Development
- **Installation**: `npm install`
- **Running Locally**: `npm run dev` starts the Next.js development server on `http://localhost:3000`.
- **Linting**: Standard ESLint configuration for Next.js is enforced via `npm run lint`.
